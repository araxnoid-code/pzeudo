## Change log
##### 0.0.1-dev.6-matmul
_20 july 2026_
- plan creation for version 0.0.1-dev.6
- added testing code for matmul_2d in the contiguous case

_20 july 2026_
- Added new testing code to matmul_2d for testing view cases.

- Supplemented the previous testing code in matmul_2d to validate f64 and f32 cases.

- Fixed a bug in OpsMatmul2DF32::matmul_2d due to an offset error.

_21 july 2026_
- Development process of pzeudo_module

- Added a new initialization method for arrays, Array::from_array

- ArrayStorage development: ArrayStorage now has a new property, update_able_tensor_storage, which stores UpdateAbleTensor.
  - UpdateAbleTensor is a struct with two properties: array(Array) and grad(Array). It functions as an array of values ​​that can be updated during training and will not be 'deleted' during training.
  - The meaning of 'deleted' is that when a tensor has performed a backward iteration and is at the end of an epoch, the storage is cleared to save the tensor calculations for the next epoch. Therefore, the Array needed by the tensor for optimization and updating is stored in another storage. 
  - Changes to ElementType include the addition of a field, ElementType::UpdateableTensor, which stores an index that points to a specific array in update_able_tensor_storage.

- Creation of an initialization method in the module, Module::new.

- Creation of tensor initialization methods in the module, including:
- Module::tensor_from_array
- Module::tensor_from_vector_with_shape

_21 july 2026_
- Added auto-broadcast to:
  - OpsAdd::add Options
  - OpsDiv::div Options
  - OpsMul::mul Options
  - OpsSub::sub Options

- Added testing code to add for auto-broadcasting cases.

- Implemented all trait view methods on ArrayRef,
  - OpsSlicing
  - OpsBroadcast
  - OpsPermute
  - OpsToShape

- Added the broadcast_detech function to pzeudo_tensor/_ops/arithmetic/_broadcast_detech, which obtains information about array broadcasts, which can be used for backpropagation purposes.

- Implement the broadcast_detect function on add, sub, div, and mul in pzeudo_tensor

- Update the RecordLabel below to store broadcast data:
  - RecordLabel::Add
  - RecordLabel::Sub
  - RecordLabel::Div
  - RecordLabel::Mul

- Handle broadcasting cases in add_backward.

_22 - july 2026_
- Implemented OpsSum and OpsAvg on ArrayRef

- Updated the following operations to handle autobroadcasting of arrays 
  - div_backward 
  - mul_backward 
  - sub_backward

- Added the matmul_2d method (for f32 and f64) to tensors

- implemented OpsMatmulF32 and OpsMatmulF64 in ArrayRef

- Added new Backward Label: 
  - BackwardLabel::Matmul2dF32 
  - BackwardLabel::Matmul2df64

- Created `matmul_2d_f32_backward` and `matmul_2d_f64_backward` to allow backward operations on matmul_2d f32 and f64, and then included them in the backward method.

- Created the StorageF32F64 trait, implemented for ArrayStorage, to combine ArrayStorage<f32> and ArrayStorage<f64>. This is because matmul_2d has two versions, specifically for f32 and f64, and the generic type cannot handle them. Therefore, the StorageF32F64 trait was created.

_22 - july 2026_
- Improved TensorTrait, which now requires no initialization for generic Contiguous or View types.

- Updated the following arithmetic methods so they now act as tensor methods, not Trait methods (trait arithmetic has been removed).
  - Tensor::add
  - Tensor::mul
  - Tensor::sub
  - Tensor::div

- Added Contiguous_Type enum:
```rs
pub enum ContiguousType {
    Arr,
    Grad,
}
```
  - Serves to differentiate Contiguous which stores arrays as Value(arr) or arrays as gradients, ContiguousType is an addition to ElementType::Contiguous:
```rs
pub enum ElementType<F> {
    UpdateableTensor(usize),
    Contiguous(Array<F>, ContiguousType), // update
    View(usize, TensorMetadata),
}
```

- Due to the addition of an enum to ElementType, there are changes to the ArrayStorage methods, particularly in the following methods:
  - ArrayStorage::get_as_array_ref
  - ArrayStorage::get_as_array_ref_mut

- Added new parameters to the ArrayStorage::get_as_array_ref and ArrayStorage::get_as_array_ref_mut methods:
```rs
    pub fn get_as_array_ref<T>(
        &self,
        idx: usize,
        arr_contiguous_type: ContiguousType, // NEW
    ) -> Result<ArrayRef<'_, F, T>, PzeudoErr> {
        // ...
    }

    pub fn get_as_array_ref_mut(
        &mut self,
        idx: usize,
        arr_contiguous_type: ContiguousType, // NEW
    ) -> Result<ArrayRefMut<'_, F, Contiguous>, PzeudoErr> {
        // ...
    }
```
  - This change is due to increased security and ensures the requested and received types.
  - This change also makes it easier to choose between array and gradient data in UpdateAbleTensor within ArrayStorage.
  - Due to the additional condition, the storage_contiguous_type_check function was created to check whether the requested ContiguousType matches the received one.

- Added Tensor::update_able_from_vector_with_shape

- Added the Linear method (pzeudo_tensor/method/linear.rs), which now has two main methods for running linear layers.
  - Linear::new
  - Linear::forward

##### 0.0.1-dev.6-epoch
_22 july 2026_
- Changed the name of update_able_tensor to permanent_tensor, as well as the storage and method names.

- Added the LinearTrait trait, which is useful for passing structs for initialization in modules via Module::new_linear.

- Implemented LinearTrait in Linear.

- Added a forward method to linear for the f64 case.

- Added new methods to Module:
  - Module::clear_storage
  - Module::permanent_tensor_from_vector_with_shape
  - Module::new_linear

- Development of Module::epoch, which can be seen in pzeudo_module/epoch.rs, which will be the core of the training loop later.

_23 july 2026_
- Added enums for storage, including:
  - StorageType
  - GetElementOutput
  - GetElementMutOutput

- Changed the ArrayStorage workflow; now you can directly access storage or permanent_storage through the StorageType enum.

- Removed ElementType::Permanent, now allowing users to directly select whether to be permanent or not through the StorageType enum.

- Internal changes and return types:
  - ArrayStorage::get_element, which will return GetElementOutput.
  - ArrayStorage::get_element_mut, which will return GetElementMutOutput.

- Changed the RecordLabel structure, which previously stored the usize, to now store the StorageType.
  - Changes were also made to functions that require access to storage through the index.

- Rewrote the ArrayStorage::get_as_array_ref and ArrayStorage::get_as_array_ref_mut methods due to changes in the ArrayStorage structure.
  - Other ArrayStorage methods that are not mentioned have also changed, but in terms of concept, workflow and usage they are still the same or not much different, that's why the others are not mentioned.

_23 july 2026_
- Added a loss function in the loss directory in pzeudo_tensor/loss.

- Added the mse_backward function.

- for backward purposes.
  - Updated RecordLabel by adding the RecordLabel::LossMse field.
  - Incorporated mse_backward into the backward method.

- Added Array::to_one and Array::to_zero.

- Added the optimizer directory to pzeudo_tensor/optimazer. The newly created optimizer is sgd.

- Updated the get_broadcast_dim function to now consistently return broadcast dimensions in order (from smallest to largest).

- Added the Module::epoch method, which will be the core of training and running AI models with pzeudo.

- Added EpochBuilder, which will be the structure used for configuration of the Module::epoch method.

- Added Module::zero_grad method.

- Implemented Debug on RecordLabel (debugging purposes only, does not provide any important information).

- For tensors that start backward, the mechanism for setting the grad of the tensor to 1 now uses the to_one method.
