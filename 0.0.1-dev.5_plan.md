## Changed Log
<!--0.0.1-dev.5-tensor-->
_16 - july - 2026_
- The process of developing a tensor consists of three main parts: tensor (contiguous and view), storage, and operations.

- Development of storage (ArrayStorage), which will store arrays (the storage property) and empty/reusable indexes (the empty_idx property).

- ArrayStorage now has the following methods:
  - ArrayStorage::new
  - ArrayStorage::get_storage
  - ArrayStorage::get_mut_storage
  - ArrayStorage::get_empty_idx
  - ArrayStorage::get_mut_empty_idx
  - ArrayStorage::push
  - ArrayStorage::remove

- Development of a contiguous tensor (Tensor), which will store array indexes (the array itself will be stored in ArrayStorage), gradient indexes (the gradient itself will still be an array), and the Reference Count (RC) of ArrayStorage itself.

- Tensor now has one method. new
  - Tensor::new

- Combine the error files into one main error file (PzeudoErr) in pzeudo/error.rs.

- Rename the tensor to pzudo_tensor.

_17 - july - 2026_
- Added a new initialization method for Arrays, namely Array::zeros

- added a new initialization method to tensors, namely Tensor::from_vector_with_shape

- implemented Display on Array and ArrayView

- added a new method to the array view

- Separate Metadata into a separate file, namely at /pzeudo_num/metadata.rs

- added ArrayRef, working as a bridge between Tensor and Array operations. has a structure similar to ArrayView but the metadata itself is not an own type but a reference, this is useful for creating a preview data type which aims to connect Tensors which will have 2 modes (contiguous and view)

- For ArrayRef, it has a generic type in the form of T, which can be filled with Contiguos and Views. dalam implementasi metode pada pzeudo_num/_ops, generic type ini lah yang akan membdakan implementasinya jika ingin ada optimalisasi.

- added a new tensor method for initialization namely Tensor::from_vector_with_shape

- ArrayStorage development

- Proses pembauatan Alur Forward, menggunakan RecordLabel untuk merekam setiap operasi yang dilakukan.

- the process of creating arithmetic methods on tensors

_18 - july - 26_
- Making only one Tensor (view and contiguous will depend on the Array via View and Contiguous).

- Added Add, Div, Mul. and Sub methods.

_18 - july - 26_
- Added ArrayStorage::get_element method

- Added PzeudoErr::StorageGetErr

- The metadata_to_array_ref function in pzeudo_tensor/_ops/tools now returns Result

- Added error handling to functions that previously used metadata_to_array_ref
  - add
  - sub
  - mul
  - div

_18 - july - 26_
- Tensors will now be split into two: Tensor and TensorView.

- Added TensorTrait trait, which allows Tensor and TensorView to operate in the same method.

- Added TensorType enum, which serves as one of the returns in TensorTrait::get_array_data, which differentiates how Tensor and TensorView are treated.

- Changed metadata_to_array_ref function, which now accepts TensorType.

- ArrayStorage will now store Arrays directly.

- Changes to tensor methods (add, sub, div, mul) due to changes in the storage concept of ArrayStorage.

_18 - july - 26_
- Adding the ElementType enum to the pzeudo_tensor/storage/element_type.rs folder has the form:
```rs
pub enum ElementType<F> {
    Contiguous(Array<F>),
    View(usize, TensorMetadata),
}
```
  
- Changed the data type stored by ArrayStorage from Array to ElementType, which also stores metadata.

- The ElementType enum has two values, including their functions in ArrayStorage.
  - ElementType::Contiguous, which stores arrays directly.
  - ElementType::View, which stores metadata and points directly to the parent array.

- ArrayStorage has undergone changes and new methods have been added:
  - Added ArrayStorage::get_as_array_ref method.
  - Changed the name of ArrayStorage::get to ArrayStorage::get_element.

- The metadata_to_array_ref function has been removed as it is no longer needed.

- Tensor has been reintegrated into the generic type that differentiates them (Contiguous and View).

- TensorTrait has been reinstated.

- Changes to Tensor arithmetic due to changes to the ArrayStorage structure.

_19 - july - 26_
- Added ArrayRefMut (as a mutable type of ArrayRef) and MutMetadata (as a mutable version of Metadata)

- Added ArrayAssignTrait trait to pzeudo_num/array/_ops/assign/assign.rs, which is a subtrait of ArrayTrait. Subtraits of ArrayAssignTrait include:
  - OpsAddAssign
  - OpsAddAssign::add_assign

- Implementing ArrayAssignTrait on Array and ArrayRefMut for ArrayView and ArrayRef will not be possible because the type they store is &[F].

- Contiguous and View in array_ref are now combined into one file, array_ref_type

- Added OpsUnary::neg method

- Added a new method to ArrayStorage, ArrayStorage::get_as_array_ref_mut, which returns ArrayRefMut.

- Defining the architecture so that value updates are not supported in pzeudo_tensor will break the backpropagation flow.

- Adding backward methods for each arithmetic tensor operation, including:
  - add_backward
  - sub_backward
  - mul_backward
  - div_backward
